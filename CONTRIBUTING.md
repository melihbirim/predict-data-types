# Contributing to predict-data-types

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/predict-data-types.git`
3. Install dependencies: `npm install`
4. Run tests: `npm test`
5. Run linter: `npm run lint`

## Making Changes

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Add tests for new functionality
4. Ensure all tests pass: `npm test`
5. Ensure code passes linting: `npm run lint`
6. Commit your changes with a clear message
7. Push to your fork and create a pull request

## Code Style

- We use ESLint for code style enforcement
- Use single quotes for strings
- 4-space indentation
- Add JSDoc comments for new functions
- Write tests for new features

## Testing

- Add test cases for any new functionality
- Ensure all existing tests continue to pass
- Test edge cases and error conditions

## Pull Request Process

1. Update the README.md if needed
2. Ensure your code is well-tested
3. Make sure CI passes
4. Request review from maintainers

## Issues

- Use GitHub issues for bug reports and feature requests
- Provide clear reproduction steps for bugs
- Include expected vs actual behavior

Thank you for contributing!
